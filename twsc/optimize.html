效能調校(DB, API, BGSVC)

DB

HFS 高速檔案系統取樣工作，資料每15分鐘產生一次，一位使用者一天就會有96筆資料。當計算完平均後，還需將計算結果透過 API 轉送。透過使用 Index 與 Partition Table 調校，將整體時間由 5小時縮短至 1小時完成

大表欄位異動

千萬級別以上資料表如要異動欄位屬性(欄位、更名、異動索引等)會造成效能瓶頸，所以開發移轉工具，可大幅提高異動欄位效率。直接異動資料表屬性可能會耗時數十分鐘，使 用工具移轉幾乎是瞬間完成(不包含移轉資料的時間，對於效能瓶頸，移轉資料的時間可 忽略)

API

一項服務如需同時取得數個 API 資訊時，可透過併發方式取得資料

Background service

背景程序時常處理非即時但複雜且耗時的任務，配合狀態機與 worker, monitor 機制 (Parallel processing)，可縮短任務消耗時間。
