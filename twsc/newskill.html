新技能(Docker, Laravel, Nginx, Linux, Percona Cluster DB)

Docker

對於開發環境，Docker絕對是個好幫手。Docker解決了開發者環境版本不同步的難題。再來就是簡化了部署的流程，我們正在將所有服務都容器化(API server side, background service, proxy..)。第三則是版本切換，很多軟體是不能在同一台機器上安裝兩種以上的版本，Docker很好的解決了這個難題，他與VM還是有不同之處(效能、效率、工作方式..)。

Laravel

前一家任職的公司，MVC框架是自行研發的，所以對MVC架構有一定的認識，但就是沒有深入研究Laravel。後來在本公司面試時，主管有說明Laravel是必要技能，我坦白沒有實際開發經驗，但對於MVC架構有一定理解。 錄取之後，這個技能我須自行熟練起來。我大約花了半個月的零碎時間先將基礎打好，後來就接到許多Laravel 開發API的工作，到目前已經有一定的熟悉度。

Nginx

因開發上需要，我們大量使用代理伺服器去控管路由設定，且區分ingress, egress於不同台主機上。如此能更好區分request 方向，且debug 也比較方便。 所以對Nginx 一些基礎設定有一定認識，而這也是必要技能。

Linux

我們部門負責的Production 主機就有3、40台以上(有區分不同的營運商)，所以基本的Linux 操作也是必要技能。

Percona Cluster DB

雖然我不是主要負責DB的人，但偶而的例行公事(災難復原、Staging rotate)還是會接觸到，所以對資料庫叢集也有一點認識。

其他

其他像是 RabbitMQ(我主要負責項目), Vue nuxt3, Redis, Ansible 等等，都是在本公司任職期間接觸熟悉的。
